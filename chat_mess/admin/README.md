# DeFensy · Admin Console

Веб-панель администратора/модератора для чат-платформы DeFensy. Позволяет в **реальном
времени** видеть **все** переписки: личные чаты, группы и каналы — а также управлять
пользователями (mute) и удалять сообщения.

Это отдельное React.js (Vite) приложение рядом с `web/` (клиент) и `back/` (сервер).

## Возможности

- **Обзор** — живые метрики (пользователи, онлайн, группы, каналы, число сообщений).
- **Прямой эфир** — единый поток всех сообщений всей платформы по мере их отправки.
- **Личные чаты** — список всех DM-диалогов + чтение полной переписки любых двух людей.
- **Группы / Каналы** — список всех групп и каналов + чтение всей истории сообщений.
- **Пользователи** — таблица аккаунтов, статус (онлайн/2FA/ghost/предупреждения),
  заглушить (mute) на 1 час / снять mute.
- **Модерация** — мягкое удаление любого сообщения (DM / группа / канал) прямо из треда.

## Как это работает

- REST: `GET /api/admin/*` — выдаёт данные из БД (текст расшифровывается на сервере).
- Real-time: при входе админский Socket.IO отправляет `admin:join` с ключом; сервер
  подписывает сокет на **все** комнаты `user_*`, `group_*`, `channel_*`, поэтому админ
  получает ровно те же события (`messageReceived`, `groupMessageReceived`,
  `channelMessageReceived`, удаления и т.д.), что и обычные клиенты.
- Групповые сообщения приходят protobuf-бинарём и декодируются `src/proto/groupMessage.js`.

## Авторизация

Все admin-эндпоинты и `admin:join` защищены ключом `x-admin-key`.

- Значение по умолчанию: `defensy-admin-2026`
- Переопределяется переменной окружения `ADMIN_KEY` в `back/.env`.

Введите этот ключ на экране входа.

## Запуск

```bash
# 1) backend (если ещё не запущен) — из chat_mess_3_3_2/back
node app.js          # :3000  (перезапустите, чтобы подхватить новый admin-роут)

# 2) admin console — из chat_mess_3_3_2/admin
npm install
npm run dev          # http://localhost:5180
```

При другом адресе бэкенда задайте `VITE_API_URL` (см. `src/config.js`).
